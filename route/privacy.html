About This Application

This platform is a personal portfolio and web application developed by Alex Legacy. It is designed to showcase projects, demonstrate technical skills, and provide interactive features such as authentication, messaging, and content management. The purpose of this application is strictly educational, developmental, and demonstrational.

The system includes user authentication, profile management, project uploads, and communication features. These features are implemented to simulate real-world applications and to demonstrate full-stack development capabilities, including frontend design, backend architecture, database management, and secure API communication.

This application is not intended to impersonate any organization, financial service, or third-party platform. It is a standalone personal project created for learning, portfolio presentation, and technical demonstration.

Purpose of the Platform

The goal of this platform is to provide a structured and interactive environment where users can explore projects, test features, and understand how modern web applications function. It serves as a demonstration of skills in areas such as:

- Full-stack web development (HTML, CSS, JavaScript, Node.js)

- Database integration and management

- Authentication systems and security practices

- UI/UX design and responsive layouts

- File uploads and cloud storage integration

Data Usage & Privacy Policy

This application respects user privacy and is committed to protecting any data that may be provided during usage.

1. Data Collection: This platform may collect basic user information such as email addresses, usernames, and profile data strictly for authentication and application functionality.

2. Data Usage: Collected data is used solely for the operation of the application, including login, personalization, and feature access. No data is sold, shared, or distributed to third parties.

3. Data Security: Reasonable security measures are implemented to protect user data, including token-based authentication and secure backend processing. However, as this is a development project, users are advised not to use sensitive or real-world critical credentials.

4. Cookies & Tokens: Authentication tokens may be stored locally on the user's device to maintain session state and improve user experience.

5. Third-Party Services: This application may use external services such as hosting platforms and databases. These services operate under their own privacy policies.

6. User Responsibility: Users are encouraged to use non-sensitive information when interacting with this platform, as it is intended for demonstration and testing purposes.

7. Updates: This policy may be updated as the application evolves. Continued use of the platform indicates acceptance of these terms.

Security Statement

This application implements standard security practices including authenticated routes, protected endpoints, and controlled access to administrative features. Any attempt to misuse the platform is discouraged and may be restricted.
